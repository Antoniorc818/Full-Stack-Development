# Full-Stack-Development
Compilation of my CS-465 Full Stack Development work



The Travlr Getaways project is a full stack online application that consists of an administrative single-page application (SPA) and a customer-facing interface. I worked with a variety of frontend and backend techniques during the development process, and in the finished version, I added secure authentication. I was able to comprehend the connections and operations between the various layers of a full stack application thanks to this project.
Two different frontend development approaches were used in this project. Express and server-rendered HTML were employed in the customer-facing section. This method sends completely rendered pages to the browser after the server creates the HTML views. The application logic is located on the server, and each user interaction usually causes a complete page reload. This approach works well for conventional web apps since it is organized and simple.
On the other hand, Angular was used to create the administrative side as a single-page application. The SPA dynamically changes content within the browser without requiring complete page reloads, in contrast to server-rendered HTML. It uses modular, reusable components to organize the user interface and makes RESTful API calls to the backend. This method offers a more effective and engaging experience. Compared to traditional rendering, the SPA greatly increased scalability, responsiveness, and maintainability, but it also required more planning and architectural organization.
A NoSQL MongoDB database was used for the backend due to its adaptability and compatibility with JavaScript-based technologies. MongoDB naturally integrates with Node.js and Express because it stores data in documents that resemble JSON. Because the frontend and backend of the application communicate data in JSON format, MongoDB's document-based structure made development easier and permitted flexible schema modifications all the way through the project.
In order to connect the frontend and backend layers, JSON was essential. Despite being based on JavaScript object syntax, JSON is only a data format and lacks executable code and logic. JSON acted as the communication link in this application. The Angular frontend requested travel information and sent login credentials in JSON format. After processing the request, the Express backend provided JSON replies. Documents were then saved and retrieved in a JSON-like manner by MongoDB. All application layers were able to integrate seamlessly because to this uniform data format.
I made many code changes during development to increase functionality and efficiency. I standardized RESTful endpoint formats, divided controller logic from route definitions, and arranged business logic into service files. Rather than rewriting user interface code, I made reusable Angular components for the frontend. These reusable parts maintained uniform design, decreased duplicates, enhanced maintainability, and allowed the setting up of future improvements. Refactoring not only improved performance but also strengthened the organization and clarity of the codebase.
An essential part of the complete stack development process was testing. I worked with API endpoints to retrieve and modify data using HTTP methods including GET, POST, PUT, and DELETE. To guarantee precise data handling, appropriate status codes, and right answers, each endpoint needed to be validated. Testing became more difficult when security and authentication elements were included. Token verification was necessary for protected routes, and appropriate error messages had to be returned for unauthorized access. My understanding of backend security was strengthened by putting middleware and password hashing into practice for route protection. Testing verified that security and functionality were functioning properly.
In summary, this course made a substantial contribution to my professional development objectives. It helped me realize how frontend and backend concepts work together inside a whole system, going beyond my comprehension of them separately. I gained practical expertise in MongoDB integration, Angular SPA architecture, RESTful API development, secure route protection, and authentication implementation. Also, I improved my debugging and refactoring abilities. I gained a greater understanding of secure application design and full stack architecture. My confidence in creating safe and scalable web applications has grown as a result of this.
